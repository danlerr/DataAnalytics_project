# Project Work Guidelines (Data Analytics and Data Driven Decision)

## Project work structure

The work must follow this structure:

1. **Description of the dataset**
2. **Data cleaning** (if needed)
3. **Exploratory analysis**
4. **Main analysis**: objective and methods adopted
5. **Preview/summary of the results**
6. **Detailed results**
7. **Conclusions**

---

## The two parts of the project

The project work is composed of **two parts**:

### a) The presentation
- Duration: **15 minutes**
- **All group members** must present one part of the project
- Slides in PowerPoint or PDF
- **No more than 15 slides**

### b) Supplementary materials and methods
- Output of the elaboration: **Python Jupyter notebook**
- Technical but sufficiently self-explained to be readable and comprehensible (with comments, etc.)

---

## Important points

- **Link presentation ↔ supplementary**: tables, conclusions and analyses contained in the presentation must point to the material in supplementary (b).
- **Size of part (b)**: it is related to the type of project, there is no specific requirement, but it must be of the right size.
- **Organization**: the split of project work must be organized by each group with the teachers.
- **Submission**: material must be sent a few days before the exam.
- **Clean and working code**: scripts and code may be run by teachers, so send clean and working code.
- **Only methods seen in the course**: only methods/models and packages seen in the course are allowed.
- **Justify your choices**: students must justify their project choices in the presentation and in the discussion.
- **Real-time experiments**: real-time experiments on the code can be asked during the presentation, so take your code ready for running during the discussion.

---

## Exam rules and evaluation

- The project can be presented only when **all group members have passed the written exam**.
- If all members have passed the written exam, they can present the project even if one member wants to repeat the written exam in a next call in order to improve the grade.
- Project work is **mandatory**.
- If the quality of the project is **too poor**, it must be improved to pass the overall exam.
- If the quality of the project is **sufficiently good**, a bonus on the final grade is obtained by the group members.
